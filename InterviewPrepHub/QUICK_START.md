# Quick Start Guide

**Your Daily Workflow for 100-Day Prep**

---

## Every Morning (Start Your Day)

### Step 1: Open Today's File (2 min)
```
Today is Day 1: Jan 20, 2026
Open: daily-problems/Day-01_2026-01-20.md
```

The file name format: `Day-XX_YYYY-MM-DD.md` where XX is day number (1-100)

### Step 2: Follow the Structure (2.5-3 hours)

Your daily file has 4 sections:

#### 🧠 DSA Practice (60 min)
1. **Study (15 min):** Read the pattern/concept explanation
2. **Problems (45 min):** Solve 2-3 LeetCode problems
   - Try solving WITHOUT looking at hints
   - If stuck after 20 min, review approach
   - Code your solution
   - Mark checkbox when done: `[x] Solved`

#### 💻 Backend Learning (45 min)
1. Read the concept explanation
2. Check the provided resources/links
3. Practice with code examples
4. Complete the learning checklist items

#### 🚀 Project Work (30 min)
1. Review today's project task
2. Implement incrementally
3. Test your changes
4. Commit to git

#### 📝 Daily Review (15 min)
1. Check off completed tasks
2. Write 2-3 key takeaways
3. Preview tomorrow's topics
4. Note any difficulties for later review

## Tips for Success

### Do This:
- ✅ Solve problems yourself first (no hints for 15-20 min)
- ✅ Understand WHY a solution works, not just HOW
- ✅ Build your project incrementally every day
- ✅ Take breaks during your 2.5-3 hour session
- ✅ Track weak areas for spaced repetition

### Don't Do This:
- ❌ Skip directly to solutions
- ❌ Batch multiple days together
- ❌ Skip project work "to catch up later"
- ❌ Ignore review sections
- ❌ Compare yourself to others

## Staying Motivated

### Weekly Wins
Every week you'll see tangible progress:
- Week 1: Solve your first 15 problems
- Week 2: Build a working REST API
- Week 4: Add database and auth
- Week 8: Deploy microservices
- Week 14: Interview ready!

### When You Get Stuck
1. Take a 5-minute break
2. Read the problem again carefully
3. Try a simpler example on paper
4. If still stuck after 20 min, check approach hints
5. Implement and understand the solution
6. Revisit the problem in 1 week

## FAQ

**Q: What if I miss a day?**
A: Don't try to "catch up" by doing 2 days. Just continue from where you left off. The dates are guidelines.

**Q: Can I skip project work?**
A: No. The project is crucial - it's proof you can build real systems for interviews.

**Q: What if a problem takes longer than allocated time?**
A: That's okay. Better to solve 2 problems well than rush through 3.

**Q: Should I use solutions/hints?**
A: Try for 15-20 minutes first. If genuinely stuck, yes, but understand WHY it works.

**Q: How do I track Atlassian-specific questions?**
A: They're integrated into weeks 3-12. You'll see them marked in daily files.
